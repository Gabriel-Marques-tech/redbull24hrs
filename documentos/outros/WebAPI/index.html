Autenticação

Gerencia o registro e o acesso de Gerentes e Auditores. Usa JWT (access token de curta duração + refresh token rotacionado). Atende RF027.

POST /auth/register/manager Cadastra um novo Gerente
RF027

Cria um usuário com perfil manager. Senha armazenada com bcrypt (10 rounds). Não requer autenticação prévia.

Headers

NomeValorObrigatório
Content-Typeapplication/jsonSim

Request Body

{
  "name":     "Pedro Gerente",
  "email":    "pedro@example.com",
  "password": "Senh@Forte123"
}
CampoTipoObrigatórioValidações
namestringSimNão vazio
emailstringSimNão vazio; unicidade verificada no banco
passwordstringSimNão vazio; criptografada com bcrypt (10 rounds)

Response — 201 Created

{
  "id":    1,
  "name":  "Pedro Gerente",
  "email": "pedro@example.com"
}

Status Codes

CódigoSignificadoQuando ocorre
201CreatedGerente cadastrado com sucesso
400Bad RequestAlgum campo obrigatório ausente ou inválido
500Internal Server ErrorErro de banco (ex.: e-mail duplicado)

POST /auth/login Autentica usuário e emite tokens JWT
RF027

Valida credenciais e retorna access token (curta duração) e refresh token (rotacionado). O campo role deve indicar o perfil pretendido — auditores inativos (is_active = false) têm o acesso negado.

Headers

NomeValorObrigatório
Content-Typeapplication/jsonSim

Request Body

{
  "email":    "ana@example.com",
  "password": "Senh@Forte456",
  "role":     "auditor"
}
CampoTipoObrigatórioValidações
emailstringSim—
passwordstringSim—
rolestringSimDeve ser "manager" ou "auditor"

Response — 200 OK

{
  "user": {
    "id":    "1",
    "name":  "Ana Auditora",
    "email": "ana@example.com",
    "role":  "auditor"
  },
  "tokens": {
    "accessToken":  "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}

Status Codes

CódigoSignificadoQuando ocorre
200OKCredenciais válidas; tokens emitidos
400Bad RequestRole inválido: { "error": "Perfil inválido. Use 'manager' ou 'auditor'" }
401UnauthorizedCredenciais incorretas ou auditor inativo: { "error": "Credenciais inválidas" }

PATCH /events/:id/pause Pausa a competição (congela o cronômetro de 24h)

Marca paused_at no evento, congelando o cronômetro de 24h sem encerrar a competição. Enquanto pausado, auditores ficam bloqueados de salvar turnos e checkpoints. O evento precisa estar in_progress e não pode já estar pausado. Acesso restrito a manager (requireAuth + requireRole("manager")).

Auditoria — Turnos

Controla o ciclo de vida dos turnos dos corredores durante a competição: início, registros de checkpoint e encerramento. Atende RF007–RF019, RF025, RF032, RF033. Todos os endpoints exigem autenticação JWT.

POST /audit/shifts/start Inicia um turno
RF007 RF008 RF009 RF010 RF011

Registra o início de um turno. Rejeita a operação se o corredor já possuir turno em aberto (RF007) ou se a esteira estiver ocupada (RF008). O start_at é gerado automaticamente pelo servidor (RF011).
